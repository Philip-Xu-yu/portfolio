# 我用AI做了一个自媒体起号助手，10分钟生成30天内容规划

> 案例拆解：AI自媒体起号助手
> 发布日期：2026-06-12

---

## 背景

朋友想做小红书，但每天为选题发愁。她说："我花了3天都没想好第一篇发什么。"

我用Claude Code花4小时做了一个AI自媒体起号助手。输入领域，10分钟生成账号定位、30天内容规划、爆款选题和完整文案。

---

## 解决的问题

| 问题     | 之前        | 之后           |
| -------- | ----------- | -------------- |
| 账号定位 | 纠结1周     | 5分钟生成      |
| 选题规划 | 每天想2小时 | 10分钟生成30个 |
| 文案写作 | 每篇2-3小时 | 5分钟生成初稿  |

---

## 技术实现

```
后端：FastAPI
AI：OpenAI GPT-4o-mini
前端：原生HTML/CSS/JS
部署：Render
```

核心功能：

1. 账号定位生成器 — 输入领域，输出人设、内容方向、变现路径
2. 爆款选题生成器 — 输入热点，输出10个选题+角度+预估热度
3. 完整文案生成器 — 输入选题，输出标题+正文+标签

---

## 使用场景

- 自媒体新人：不知道发什么 → 用定位功能找到方向
- 内容创作者：没选题灵感 → 用选题功能批量生成
- 运营人员：写文案太慢 → 用文案功能快速出稿
